# 04-issue-comment-context

## Цель
Сделать сбор контекста Issue/PR централизованным и корректным: только актуальные, не скрытые и не резолвнутые замечания.

## Что делаем и где
- Реализуем сбор комментариев и review‑тредов в codexctl через GitHub API (GraphQL).
  - Файлы: `codexctl/internal/github/*` (новый пакет), `codexctl/internal/cli/prompt.go`.
- Расширяем контекст шаблонов для промптов.
  - Файлы: `codexctl/internal/config/config.go`, `codexctl/internal/prompt/*`.
- Обновляем шаблоны промптов, чтобы в них попадали только:
  - Issue‑комментарии: не бот, не minimized.
  - PR‑треды: не resolved, комментарии не minimized.
  - Файлы: `codexctl/internal/prompt/templates/*`.
- Добавляем инструкции с ID комментариев, чтобы агент отвечал точечно на каждое замечание.

## Зачем
- Устраняем повторные ответы на закрытые темы.
- Агент видит только актуальные и требующие реакции комментарии.
- Упрощаем работу с обратной связью через единый промпт.

## Ожидаемый результат
- В промпте отображаются только «живые» замечания.
- Агент отвечает на них корректно и адресно.
