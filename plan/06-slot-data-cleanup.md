# 06-slot-data-cleanup

## Цель
Удалять данные слотов (Postgres/Redis/кэш) при очистке окружений, чтобы не переносить старое состояние в новые задачи.

## Что делаем и где
- Добавляем очистку директории слота при cleanup и GC.
  - Файлы: `codexctl/internal/cli/manage_env.go`.
- Используем `DATA_ROOT` для вычисления пути данных слота.
  - Контракт: `DATA_ROOT/slots/<slot>` для ai‑слотов.
- Обновляем документацию о поведении очистки.
  - Файлы: `project-example/README_RU.md`, `Alimentor/README_RU.md`.

## Зачем
- Исключить наследование старой БД при повторном использовании слота.
- Повысить предсказуемость поведения окружения.

## Ожидаемый результат
- При TTL‑очистке и ручном cleanup удаляются данные слота.
- Новый запуск слота получает чистую среду.
