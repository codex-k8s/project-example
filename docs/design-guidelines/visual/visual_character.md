# Визуальный характер и база

Документ задаёт общие требования к визуальному стилю и базовой композиции.

## 1) Визуальный характер

**Цель:** интерфейс должен быть спокойным, функциональным и не отвлекать от работы.

- Светлая тема по умолчанию: белый/почти белый фон, мягкие поверхности, тонкие разделители.
- Минимум декоративности: никаких лишних градиентов, текстур, пёстрых иллюстраций, тяжёлых теней.
- Один главный акцент (брендовый цвет) + семантические цвета (успех/ошибка/предупреждение/инфо).
- Много воздуха: свободное пространство — часть дизайна, а не «пустота».

## 2) Сетка и композиция

Используй предсказуемую структуру и повторяемые шаблоны.

Базовые паттерны страниц:
- Продуктовая консоль: левый сайдбар + верхняя полоса (опционально) + основной контент.
- Документация: левый сайдбар (оглавление) + основной контент + правый TOC/якоря.
- Онбординг/пустое состояние: центрированная герой‑зона (заголовок + краткое описание + основное действие) + подсказки.

Ширины и выравнивания:
- Максимальная ширина читаемого контента в документации должна быть комфортной для чтения.
- В аналитике/логах допускается широкий контент, но с фиксированными колонками,
  горизонтальной прокруткой и «липкими» заголовками таблиц.

## 3) Типографика

**Принцип:** текст — главный UI‑компонент.

- Используй системный шрифт.
- Иерархия:
  - H1: крупный, короткий.
  - H2/H3: компактные подзаголовки, помогают сканировать страницу.
  - Основной текст: средний размер, высокий контраст, комфортный межстрочный интервал.
- Длина строк для абзацев — умеренная, чтобы читать без усталости.
- Не злоупотреблять жирным: оно для акцентов, а не для «разукрашивания».

## 4) Цвет, контраст, состояния

Палитра:
- Фон: белый/очень светлый нейтральный.
- Поверхности: белый + лёгкое отличие от фона (через тон/границу).
- Разделители: тонкие и нейтральные.
- Акцент: один (брендовый). Используй для активной навигации, primary‑действий,
  ключевых метрик/выделений на графиках.

Семантика:
- Успех — зелёный.
- Ошибка — красный.
- Предупреждение — жёлтый/оранжевый.
- Инфо — синий/нейтральный акцент.

Контраст и читаемость:
- Текст должен читаться на всех состояниях (normal/hover/selected/disabled).
- Не кодируй смысл только цветом: добавляй иконку/лейбл/текст.

## 5) Пространство, радиусы, тени

Ритм отступов:
- Используй шкалу 4/8 (4, 8, 12, 16, 24, 32…).
- Вертикальные интервалы в списках/таблицах — ровные и повторяемые.
- Внутренние отступы карточек — ощутимые, чтобы контент «дышал».

Скругления:
- Карточки/панели: умеренные радиусы.
- Чипы/пилюли/инпут «в стиле командной строки»: более округлые (до pill).

Тени:
- Минимальные, едва заметные. Предпочтение: границы и лёгкое поднятие,
  а не «тяжёлая» тень.

## 6) Адаптивность

- Интерфейс обязан быть адаптивным: ключевые сценарии доступны на узких экранах без горизонтального скролла страницы.
- На мобильных экранов переходи к одно‑ или двухколоночным макетам, не сохраняя desktop‑плотность.
- Второстепенные элементы допускается переносить в меню/оверлеи, но primary‑действие должно оставаться видимым.
- Горизонтальный скролл допустим только внутри таблиц/логов, а не на уровне всей страницы.
