# Состояния, доступность, контент

## 1) Пустые состояния и онбординг

Когда данных нет — интерфейс всё равно должен быть полезен.

- Центрированный заголовок + короткий текст «что делать дальше».
- Одна большая primary‑кнопка.
- Ниже — 2–4 шаблона/подсказки/быстрых сценария.
- Не использовать «маркетинговые» абзацы.

## 2) Статусы, бейджи, чипы

- Статусы показывай бейджами‑пилюлями (нейтральная подложка + цветной текст/иконка).
- Чипы используй для фильтров, быстрых действий, отображения контекста (проект/ветка/режим).

## 3) Ошибки, предупреждения, уведомления

Ошибка должна отвечать на:
1. что произошло,
2. почему (если известно),
3. что делать дальше.

Тон — нейтральный. В логах/таблицах ошибки показывай компактно
(бейдж + возможность раскрыть детали).

## 4) Иконки

- Один стиль на весь продукт (линейные, одинаковая толщина).
- Иконка помогает распознать действие, но не заменяет текст.
- Не ставить иконку «ради красоты».

## 5) Движение и интеракции

- Анимации минимальные, короткие, функциональные: hover, раскрытие деталей,
  переключение табов/фильтров.
- Без «прыжков» layout’а: размеры элементов не должны резко меняться.

## 6) Доступность

- Фокус‑контуры видимы (особенно на интерактивных элементах).
- Кликабельные зоны достаточного размера.
- Любое действие с иконкой — с доступным label/tooltip.
- Таблицы и фильтры должны быть управляемы с клавиатуры.

## 7) Touch и жесты (mobile)

- Все интерактивные элементы должны быть удобны для большого пальца.
- Критичные кнопки — в зоне досягаемости снизу/справа.
- Мелкие иконки в таблицах заменяй на меню «⋯» строки.
- Увеличивай hit‑area у иконок, особенно в app bar и в списках.
- Не полагайся на «точные» клики.

## 8) Текст, переносы, заголовки на mobile

- Заголовки короче, чем на desktop; допускают перенос без «висячих» слов.
- Второстепенные подписи — в одну строку с обрезкой, полное значение — по раскрытию.

## 9) Состояния загрузки и производительность на mobile

- Используй skeleton для списков/карточек.
- Фиксируй высоты ключевых блоков (app bar, композер, карточки метрик).
- Избегай «тяжёлых» экранов: виртуализация списков/логов, пагинация или «загрузить ещё».

## 10) Контент и микро‑копирайт

- Короткие заголовки, ясные подписи, минимум воды.
- Единый стиль регистра: пункты меню — существительные, действия — глаголы.
- Для RU‑локализации:
  - аккуратные кавычки «ёлочки» при необходимости;
  - даты/время в одном формате;
  - избегать англицизмов, если есть простая замена.
