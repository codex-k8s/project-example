# Visual Design Guidelines

Документы задают требования к визуальному стилю и UI‑паттернам для продуктов в духе современной SaaS‑консоли.

Стартовая точка:
- `docs/design-guidelines/visual/visual_character.md` — визуальный характер, сетка, типографика, цвет, отступы.

Остальные документы:
- `docs/design-guidelines/visual/navigation_inputs_actions.md` — навигация, prompt‑bar, кнопки и иерархия действий.
- `docs/design-guidelines/visual/components_data.md` — карточки, таблицы/логи, графики, документация и код‑блоки.
- `docs/design-guidelines/visual/states_accessibility_content.md` — пустые состояния, статусы, ошибки, иконки, движение, доступность, микро‑копирайт.
- `docs/design-guidelines/visual/check_list.md` — чек‑лист качества для агента.
- `docs/design-guidelines/visual/response_format.md` — рекомендуемый формат ответа агента при описании UI.

Связано с frontend‑гайдами: `docs/design-guidelines/vue/AGENTS.md`.
