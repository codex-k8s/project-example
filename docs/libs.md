# Общие библиотеки и переиспользуемая логика

В этом примере проекте общие библиотеки сведены к минимуму,
чтобы сконцентрироваться на демонстрации стека `codexctl + K8s`.

При добавлении переиспользуемой логики придерживайтесь следующих правил:

1. **Go‑библиотеки**
   - Располагайте их в `libs/go/<name>`.
   - Экспортируйте только то, что действительно нужно нескольким сервисам
     (например, обёртки для логгера, метрик, HTTP/GRPC‑серверов).
   - В сервисах импортируйте пакеты из `github.com/ВАШ_АККАУНТ/project-example/libs/go/...`.

2. **Python/Django**
   - Общий код выносите в отдельные модули/пакеты в `libs/python` (если потребуется)
     и подключайте через `PYTHONPATH` или обычный импорт.

3. **Фронтенд**
   - Переиспользуемые утилиты и хуки Vue выносите в `services/web_frontend/src/lib`
     или `services/web_frontend/src/composables`.

4. **Документация**
   - При добавлении новой библиотеки обновляйте этот файл и кратко описывайте
     назначение и место использования.

В текущей версии:

- Go‑сервис `chat_backend` использует только стандартные библиотеки и
  внешние зависимости (`pgx`, `go-redis`, `bcrypt`) внутри собственного пакета;
- Django‑сервис и фронтенд также не имеют вынесенных общих библиотек.
